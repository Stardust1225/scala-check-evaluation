We use 6 projects in our empirical study, and collecting warnings thrown by 115 checks from compiler output. The compiler output is in compiler-output.zip, in which each document represent output from a release version, and the number in the filename represents the sha of the release commit gengrated by git. The source code is in data.zip.

To extract warnings from compiler output, we use two regular expressions for the 115 checks, which are "\[warn\]\s.\.scala:[0-9]:\s\[[a-zA-Z]\]." and "\[warn\]\s.\.scala:[0-9]:\s\[scapegoat\].*". Through the two expressions, we can match warnings in the compiler output.

Tool for finding false-positive warnings

We implement the tool as a compiler plugin, We provide the source code of the tool and tool packaged in Scala version 2.12. After downloading the tool, you can create a txt file named "warning.txt", and put one warning in the file, then use the tool as a common compiler plugin. If the warning is false-positive, the compilation would failed with an error message "This warning is wrong". To saving the insepction time, the tool will interrupt the compilation with an error message "This warning is true" if the warning is not false-positive.
